# 경제적 자유 시뮬레이터

싱글·부부 모드로 **은퇴 가능 나이**와 **FIRE 단계별 도달 시점**을 계산하는 웹앱.
소득·소비·자산·자산배분(글라이드패스)·시나리오 변수(연금·침체·자녀·목돈·둠스데이)를 넣으면 생애 현금흐름을 시뮬레이션한다.

> 이 문서는 Claude Code에 그대로 넘겨 작업을 이어가기 위한 **사양서 겸 작업 지시서**다.
> 현재 상태(완료된 Phase 1·2)와 다음 작업(Phase 3)이 모두 담겨 있다.

---

## 빠른 시작 (로컬 실행)

전제: Node.js 20+ 설치.

```bash
npm install
npm run dev      # http://localhost:5173 에서 미리보기
npm run build    # dist/ 에 정적 빌드 산출
npm run preview  # 빌드 결과 로컬 확인
```

## 배포 (GitHub Pages)

이 저장소에는 자동 배포 워크플로(`.github/workflows/deploy.yml`)가 들어 있다.

1. GitHub에 저장소를 만들고 이 폴더를 푸시한다 (브랜치: `main`).
2. **저장소 이름을 `vite.config.js`의 `base`에 맞춘다.** 예: 저장소가 `freedom-simulator`면 `base: "/freedom-simulator/"`. 이름이 다르면 그 이름으로 바꾼다. (`username.github.io` 같은 사용자 페이지나 커스텀 도메인이면 `base: "/"`.)
3. GitHub 저장소 → **Settings → Pages → Source: GitHub Actions** 선택.
4. `main`에 푸시하면 자동 빌드·배포된다. 주소는 `https://<사용자>.github.io/<저장소>/`.

> Vercel로 가고 싶다면: 저장소 import → 프레임워크 Vite 자동 인식 → 배포. 이 경우 `base`는 `"/"`로 둔다. (계정·DB 등 Phase 3C 확장에 유리.)

```
freedom-simulator/
├─ index.html
├─ package.json
├─ vite.config.js          # base = 저장소 이름 (GitHub Pages용)
├─ tailwind.config.js
├─ postcss.config.js
├─ .github/workflows/deploy.yml
└─ src/
   ├─ main.jsx             # 진입점
   ├─ index.css            # Tailwind + reduced-motion
   └─ App.jsx              # 앱 전체 (현재는 단일 파일)
```

**리팩터링 권장(Phase 3 초입):** `App.jsx`가 단일 파일이다. 유지보수를 위해 다음으로 분리하는 것을 첫 작업으로 권장:
`src/engine.js`(순수 계산: `equityWeight`, `useEngine`), `src/format.js`(`won`, 축 포맷), `src/components/`(ModeSelect, Inputs, Results, Advanced, AllocCard, Share, YearTable, charts). 기능 변경 없이 구조만 나누는 PR로 시작할 것.

## 현재 구현 상태 (Phase 1·2 완료)

### 화면
- **모드 선택**(첫 화면): 싱글 / 부부. 선택에 따라 입력·계산 분기.
- **입력 탭**: 사람별 입력 + 생활비 + 투자·경제 가정 + 자산배분 + 고급 옵션 + 시나리오 공유.
- **결과 탭**: `일반 은퇴`(예상 은퇴 가능 나이 + 자산곡선) / `단계별 FIRE`(Coast→Fat 도달나이·달성률 + 목표선 차트) 토글. 부부는 가구/개인 전환.
- **연도별 탭**: 예상 수명까지 연도별 표(소득·연금·소비·자산). 자산이 음수면 칸 비움.

### 입력 항목
- 사람별: 현재 나이, 현재 자산(=시작 자산), 연 소득(세후), 연평균 수입 증가율, 예상 정년, (나) 예상 수명, 연 비정기 지출. 연금 옵션 ON 시: 연금 개시·수령액.
- 생활비(만원, **현재가치**): 월 현재/Lean/Standard/Fat 단계 + 연 비정기 지출. (부부는 사람별, 가구=합산)
- 경제 가정: 위험자산 수익률, 생활비 증가율, 물가상승률, 인출률(SWR), 자산 매각비용.
- 자산배분: 위험자산 비중 규칙(100/110/120−나이, 고정, 라이징 본드텐트), 안전자산 수익률.
- 고급(기본 OFF): 연금, 경기 침체 주기, 자녀 양육, 일회성 목돈 이벤트, 둠스데이.
- 공유: 입력 전체를 URL 해시(`#s=<base64>`)로 인코딩 → 링크 복사 / 붙여넣기 불러오기.

### 엔진 로직 (모두 만원·연 단위, 명목)
타임라인은 '나'의 현재 나이~예상 수명. 매년:

- **소득** = 정년 전까지 `연소득 × (1+수입증가율)^t` (사람별 합산).
- **연금**(옵션) = 개시나이부터 `수령액 × (1+물가상승률)^t`. *일반 은퇴 계산에만* 반영.
- **소비** = `(월생활비×12 + 연 비정기) × (1+생활비증가율)^t`. 자녀비는 `(1+물가)^t`.
- **자산배분 비중** `w` = 규칙별 위험자산 비중(`equityWeight`).
- **블렌디드 수익률** = `w × 위험자산수익률 + (1−w) × 안전자산수익률`. 침체 해엔 위험부분이 `−하락폭`.
- **자산 다음해** = `자산 + 소득 + 연금 − 소비 + 자산×블렌디드 − 매각비용`. 매각비용 = (소비>소득인 해) 인출액 × 매각비용%.
- **목돈** = 해당 나이에 `금액 × (1+물가)^t` 차감. **둠스데이** = 해당 나이에 `자산 × (1 − w×충격%)` (위험자산만 타격).
- 자산이 음수가 되면 그 나이를 **고갈**로 기록.
- **예상 은퇴 가능 나이** = 그 나이에 모두 일을 멈춰도 예상 수명까지 자산이 안 마르는 가장 이른 나이(완전탐색).
- **FIRE 도달**: 적립 곡선(연금 제외, 자산만)이 각 단계 목표선을 넘는 첫 나이. 목표 = `연생활비(단계) / SWR`, 매년 `(1+생활비증가율)^t`로 상승. Coast = 정년에 Standard에 도달하도록 역산한 현재 필요액.

### 단위·기본 가정
- 모든 금액 **만원**. 퍼센트는 연율. 기본값: 수익률 7%, 생활비증가율 3%, 물가 2.5%, SWR 4%, 매각비용 0.5%, 자산배분 110−나이, 안전자산 3%, 예상 수명 100.

### 의도적 단순화 (Phase 3에서 보완 대상)
- **결정론적 평균수익** — 변동성·수익률 순서(시퀀스) 리스크 미반영.
- **세금·건강보험료 미반영** (매각비용만 근사).
- 부동산이 자산에 합산(유동/비유동 미분리).
- 은퇴 후 소비 일정(활동기/안정기/간병기 변화 없음).
- SWR 고정(은퇴 기간 길이 미보정), 운용보수 미반영.
- 단일 예상수명(배우자 사별 시나리오 없음).

## 작업 지시 — Phase 3 로드맵

자산관리 전문가 검토에서 도출한 보완점이다. **3A → 3B → 3C 순서**를 권장하되, 3A의 모델 정교화와 UX 개편은 병행한다.

### 3A — 신뢰도 핵심 (먼저)
- [ ] **몬테카를로 시뮬레이션** *(플래그십)*: 수익률을 평균·변동성(또는 역사적 데이터)로 추출해 1,000회+ 시뮬 → **성공확률 %**와 자산 백분위 밴드(10/50/90%) 표시. 결과 헤드라인을 단일 숫자 → "52~58세 (성공확률 85%)" 형태 범위로. 시퀀스 리스크가 여기서 드러나야 함.
- [ ] **세금·건강보험료 모듈**: 투자수익·연금소득 실효세율 + 건강보험료(피부양/지역) 근사. 인출액에서 차감.
- [ ] **운용보수(fee drag)** 입력(연 %) + **기간보정 SWR**(은퇴 기간 길수록 SWR 자동 하향: 30년 4% ↔ 50년 3.0~3.5%).
- [ ] **입력 검증·가드레일**: 정년<현재나이, 수명<정년, 음수 등 경고. 비현실 조합 안내.

### 3B — 한국형 현실 반영
- [ ] **부동산 분리**: 실거주/전세보증금(비유동) ↔ 투자자산 분리. 주택연금·다운사이징(특정 나이에 현금화) 옵션.
- [ ] **은퇴 후 소비곡선**: go-go(활동) → slow-go(안정) → no-go(간병) 3단계 + 간병비 spike.
- [ ] **국민·퇴직연금 추정 도우미**: 소득·가입기간 입력 → 예상 수령액 추정(수기 입력 대체 가능).
- [ ] **FIRE에 연금 반영 토글**: 목표 = `(연생활비 − 연금) / SWR`로 필요자산 축소 옵션.

### 3C — 플랫폼화 (공유 목표 완성)
- [ ] 정식 호스팅(커스텀 도메인) + (선택) 계정·클라우드 저장(Supabase/Firebase). URL 공유는 이미 동작.
- [ ] **시나리오 A/B 비교** 뷰(예: 정년 55 vs 60, 110−나이 vs 본드텐트).
- [ ] **결과 카드 이미지**(카카오톡 공유용) + PDF 리포트. 면책·출처 문구 포함.
- [ ] 다국어/통화 단위 옵션.

### UX/UX 원칙 (3A와 병행)
비전문가가 막막하지 않게 — **답 먼저, 복잡함은 나중에(progressive disclosure)**:
- [ ] **온보딩 위저드**: 첫 사용은 핵심 질문 4~5개만(나이·자산·월 생활비·소득) 한 화면씩 → 즉시 결과 → 이후 고급 입력은 펼침.
- [ ] **쉬움/전문가 모드 스위치**: 기본 '쉬움'(핵심 입력 + 큰 결과), '전문가'에서 현재 전체 옵션.
- [ ] **답을 크게, 근거는 접어서**: "은퇴 57세"를 먼저, '왜요?'를 눌러 표·차트 펼침.
- [ ] **불확실성 정직하게**: 몬테카를로 도입 시 범위·확률로 표시, 거짓 정밀(57.0세) 금지.
- [ ] **용어 친화**: SWR·Coast FIRE·글라이드패스에 ⓘ 한 줄 설명 + 즉시 환산("4% 인출 = 자산의 25배").
- [ ] **입력 친화**: "1.2억/5천/월 250" 자연 입력·자동 환산, 숫자패드, "모르겠어요→평균값" 버튼, 슬라이더 실시간 반영.
- [ ] **행동 넛지**: "2년 앞당기려면 매달 +35만원" 같은 실행 가능한 코칭 한 줄.
- [ ] **자동 저장·복원**(localStorage), 현재가치 기본 표시(명목은 토글).
- [ ] **품질 바닥**: 모바일 반응형, 키보드 포커스, 명도 대비, reduced-motion(이미 index.css에 반영), 친절한 빈 상태/오류 문구.
